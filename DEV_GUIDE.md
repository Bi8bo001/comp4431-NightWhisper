# 开发服务器操作指南 🚀

## 如何停止开发服务器

在运行 `npm run dev` 的终端窗口中：

1. **按 `Ctrl + C`** (Mac/Linux) 或 `Ctrl + C` (Windows)
   - 这会停止当前运行的开发服务器

## 如何重新启动

停止服务器后，再次运行：

```bash
npm run dev
```

## 快速操作流程

```bash
# 1. 停止服务器：在终端按 Ctrl + C

# 2. 重新启动
npm run dev

# 3. 查看输出，找到访问地址（通常是 http://localhost:5173 或 5174）
```

## 其他有用的命令

```bash
# 构建生产版本
npm run build

# 预览生产构建
npm run preview
```

## 端口被占用怎么办？ 🔧

如果遇到端口一直被占用的问题（即使你已经按了 Ctrl+C），可能是之前的进程没有完全关闭。

### 方法 1: 使用清理脚本（推荐）

```bash
# 运行清理脚本
./cleanup.sh
```

这个脚本会自动找到并关闭所有占用端口的进程。

### 方法 2: 手动清理

```bash
# 查找占用端口的进程
lsof -ti:5173,5174

# 如果看到进程ID，强制关闭它们
kill -9 <进程ID>

# 或者关闭所有 vite 进程
pkill -f vite
```

### 方法 3: 让 Vite 使用其他端口

如果不想清理，可以直接让 Vite 使用其他端口：

```bash
npm run dev -- --port 3000
```

## 代码没有更新？ 🔄

如果修改代码后浏览器没有自动更新：

1. **硬刷新浏览器**：
   - Mac: `Cmd + Shift + R` 或 `Cmd + R`
   - Windows/Linux: `Ctrl + Shift + R` 或 `Ctrl + F5`

2. **清除浏览器缓存**：
   - 打开开发者工具 (F12)
   - 右键刷新按钮 → "清空缓存并硬性重新加载"

3. **检查终端输出**：
   - 确保没有编译错误
   - 查看是否有 "page reload" 的提示

## 提示 💡

- 如果端口被占用，Vite 会自动使用下一个可用端口
- 修改代码后，浏览器会自动刷新（热重载）
- 不需要每次都重启服务器，除非修改了配置文件
- **重要**：如果按 Ctrl+C 后端口还是被占用，说明进程没有完全关闭，使用 `./cleanup.sh` 清理
